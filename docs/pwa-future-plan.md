# PWA Future Plan

最終更新: 2026-06-09

## 現在の方針

今はPWA本格公開を行わず、iOS App Store版を主軸にする。App Store版の反応、利用者からの要望、保守負荷を見て、Cloudflare PagesなどでWeb/PWA公開を検討する。

## PWA公開を検討する条件

- App Store版に一定の反応がある
- 家族や知人へURLで共有したい需要がある
- 自治体、図書館、学校、イベントなどでQRコード配布したい場面がある
- App Storeを使わない端末でも見たい需要がある
- サポートと更新の負荷が許容できる

## Web/PWA公開のメリット

- URLだけで共有できる
- iPhone、Android、PCからアクセスしやすい
- App Store審査を待たずに更新できる
- QRコード配布と相性がよい
- 家族、自治体、図書館、学校イベントで案内しやすい

## App Store版との違い

- App Store版は端末へインストールして使うiOSアプリ。
- PWA版はブラウザで開き、ホーム画面に追加して使えるWebアプリ。
- PWA版はブラウザ仕様やキャッシュの状態に影響を受ける。
- PWA版を公開する場合、公開URL、サポート、プライバシーポリシー、更新方針をWeb向けにも整理する必要がある。

## 配布の可能性

- 家族や知人へURL共有
- 図書館や地域施設でのQRコード掲示
- 学校や家庭学習での補助教材
- 自治体やイベントでの簡易配布
- App Store版への誘導ページとしての利用

## PWA公開時のチェック項目

- Cloudflare Pagesなどの公開先を決める
- 公開URLを決める
- サービスワーカーとキャッシュ更新挙動を確認する
- オフライン閲覧がブラウザで安定して動くか確認する
- iPhone Safari、Android Chrome、PCブラウザで表示確認する
- 画像100枚の読み込み容量と初回表示速度を確認する
- プライバシーポリシーをWeb/PWA版にも合う内容にする
- 問い合わせ先メールを正式化する
- 外部解析、広告、フォームを入れる場合はプライバシー回答を見直す
- App Store版とPWA版の違いをFAQへ追記する
- URLをQRコード化する場合、配布先と更新停止時の案内方法を決める
